Introduction

This is a guide to composing generative music with JavaScript.

To help us focus, the book has a few guiding principles:

  • Fun over rigour: We'll take inspiration from science, but take some artistic license along the way.

  • Composition over sound design: The focus will be on creating musical structures, with sound generation / synthesis delegated to software instruments.

  • Algorithms over performance: No interaction, self evolving. Rule based.

  • Simple building blocks: We'll build things from the ground up. Take our learnings from the previous chapter and wrap them into helpers, more expressive, DSL. Capture musical knowledge.

To help us get started, the book starts out with short primers on each of its three subjects:

  1. Generative: Here we define what we mean by 'generative'. We'll cover how natural processes? have been applied to programming, art, music.

  2. Music: Here we'll cover the building blocks of music, including melody, harmony and rhythm.

  3. JavaScript: Pointers on how to learn JavaScript (ES6), plus introduction to the Web Audio APIs. Why programming for the browser is a good thing. Other music languages.

With that under our belts, the main part of the book guides us through applying these ideas to making some noise, and potentially, music! Each chapter is based around exploring the musical potential of one generative idea. Each chapter builds on the last, building up a set of modules we can plug together to make more sophisticated music.

Mention basis for workshops?

What you'll need

  • A web browser that supports the Web MIDI API (I use Chrome)
  • A text editor (I use Atom)
  • Internal MIDI routing (Mac, Audio MIDI Setup)
  • DAW or soft synth where MIDI can be routed (I use Reason)

I use macOS, patches for Linux/Windows welcome.

Hello World

This example covers the basics of sending MIDI messages to a DAW via the Web MIDI API.

Single note

// `requestMIDIAccess` returns a `Promise` that we need to await, so we wrap
// everything in an async function.
(async () => {
  const midi = await navigator.requestMIDIAccess();

  // We're only interested in a single output.
  const output = midi.outputs.values().next().value;
  console.log(output.name); // Probably 'IAC Driver Bus 1' on macOS

  // On MIDI channel 1 (144), play a C note (60, or middle C) at full velocity (127).
  output.send([144, 60, 127]);

  // You'll often see messages specified in hex (base 16) notation (an old MIDI
  // convention). This is equivalent to the previous code.
  output.send([0x90, 0x3c, 0x7f]);

  // A single note is actually 2 messages, a note on (`NON`), equivalent to
  // pressing a key on the piano, and a note off (`NOF`), equivalent to
  // releasing the key.
  const note = 60; // middle C
  const noteOn = 144; // channel 1 note on
  const noteOff = 128; // channel 1 note off
  const velocity = 127; // maximum velocity
  const length = 1000; // 1 second = 1 beat at 60 bpm

  output.send([noteOn, note, velocity]);
  output.send([noteOff, note, velocity], window.performance.now() + length);
})();

Repeating notes

To repeat the note we can send the message at set intervals:

(async () => {
  const midi = await navigator.requestMIDIAccess();
  const output = midi.outputs.values().next().value;
  const length = 4000; // 4 seconds = 4 beats at 60 bpm

  setInterval(() => {
    output.send([144, 60, 127]);
    output.send([128, 60, 127], window.performance.now() + length);
  }, length);
})();

Learning

What can we take from the above code and encapsulate into our library?

  • MIDI access: We always need MIDI access, and for the purposes of this book, we only need a single output → midi.

  • Timing: Scheduling notes, and starting/stopping playback could be easier, and we'll eventually need a way to keep multiple things in sync → metronome.

  • Playing notes: Notes always use the same output, and always need a scheduled note off message → instrument

With that, we can rewrite our example to express our intentions more clearly:

import { instrument, metronome, midi } from 'gem';

midi().then(output => {
  const metro = metronome();
  const inst = instrument({ output, metro });

  metro.onTick(() => inst(60));
  metro.start();
});

While the musical results are less than inspiring, we now have a way to generate two key aspects of music: notes/pitch and rhythm/time, which combined can give us melody. With that in mind, we're ready to move on to a more interesting example.
